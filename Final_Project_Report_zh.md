### 2.1 数据预处理

【ZZ：需要示例图：原始行数据 → 处理后数据】

**图 1：** 数据处理示例

我们通过处理一批新闻文章 URL 并提取适合文本分类的结构化输入来构建数据集。每个 URL 首先被解析，得到一个**伪标题（pseudo-headline）**，作为模型的文本输入；对应标签则根据来源域名推断。

具体而言，我们从包含文章 URL 的 CSV 文件读取数据。对每个 URL，执行一系列归一化与清洗步骤：先去掉查询参数与片段标识符以得到干净 URL；再将 URL 按路径分段，并选取**最后一个有意义的片段**作为构造标题的基础。若最后一段包含无信息量的标识符（例如记录 ID 如 “rcrd”），则改用前一段。

我们进一步清洗提取出的片段：去掉文件类后缀（如 “.print”），并过滤数据集特有的痕迹（例如 NBC URL 中常见的末尾文章 ID）。剩余文本通过将连字符替换为空格、移除非字母数字字符、全部转小写、合并多余空白等步骤，转化为伪标题。

最后，根据 URL 的来源域分配标签：`foxnews.com` 上的文章标为 **FoxNews**，其余标为 **NBC**。由此得到「清洗后的文本—二分类标签」数据集，供后续训练与评估使用。

### 2.2 两阶段模型探索与改进

在本项目中，我们采用**两阶段建模**策略，系统性地探索并改进短文本分类的不同方法。目标不是依赖单一模型，而是通过结构化探索识别有效的表示与架构，再对最有希望的模型做针对性改进。

【XC：核对下列模型】

**第一阶段**，我们在不同表示范式之间做**粗粒度探索**。对每种表示类型，再实例化多种模型架构以体现不同归纳偏置，从而联合比较「表示方式」与「模型设计」。

- **稀疏表示：** TF-IDF 特征 + 线性分类器，具体为逻辑回归（Logistic Regression）与支持向量机（SVM）。
- **静态词向量：** 预训练词嵌入（GloVe）+ 神经网络，包括一维卷积网络（CNN）与多层感知机（MLP）。
- **上下文嵌入：** 基于 Transformer 的模型，包括 BERT 与 RoBERTa，并在分类任务上微调。

在每一类内部，我们尝试多种变体以捕捉不同归纳偏置。该阶段用于在根本不同的方法之间做比较，并找出各表示族中表现最好的模型。

**第二阶段**，我们聚焦对选定模型的**迭代改进**。具体地，从每个表示组中选出最佳模型，通过针对性修改与超参调优进一步提升性能，包括探索架构变体、训练策略以及各模型族特有的关键设计。通过缩小候选范围，我们能在不过度增加算力成本的前提下做更细粒度的优化。

该两阶段框架在「广度」与「深度」之间取得平衡：第一阶段在多种表示方法间做原则性比较；第二阶段在各族内部做细致优化。由此可分离「表示」与「架构」的影响，并更深入理解哪些建模选择对新闻标题分类最有效。

### 2.3 模型评估

本节在 2.1 节所述处理后数据集上评估模型性能，训练流程遵循 2.2 节的两阶段设定。

#### 2.3.1 实验配置

我们在三种表示范式下共考虑六种模型。对每个模型，定义输入表示 \(x\)、模型函数 \(f_\theta(\cdot)\)、预测概率 \(\hat{y}\) 及相应损失 \(L\)。

**TF-IDF + 逻辑回归**  
给定文本输入 \(x\)，提取 TF-IDF 特征向量 \(\mathbf{v} \in \mathbb{R}^d\)。模型为

\[
\hat{y} = \sigma(\mathbf{w}^\top \mathbf{v} + b),
\]

其中 \(\mathbf{w} \in \mathbb{R}^d\)，\(b \in \mathbb{R}\)，\(\sigma(\cdot)\) 为 sigmoid。损失为二分类交叉熵：

\[
L = -\big[ y \log \hat{y} + (1-y)\log(1-\hat{y}) \big].
\]

超参包括 TF-IDF 维度 \(d\) 与正则强度 \(\lambda\)。

**TF-IDF + SVM**  
使用同样的 TF-IDF 特征 \(\mathbf{v} \in \mathbb{R}^d\)，预测为

\[
\hat{y} = \operatorname{sign}(\mathbf{w}^\top \mathbf{v} + b).
\]

损失为 hinge loss：

\[
L = \max\big(0,\, 1 - y(\mathbf{w}^\top \mathbf{v} + b)\big).
\]

超参为正则强度 \(\lambda\)。

**GloVe + MLP**  
设词元序列为嵌入 \(\{\mathbf{e}_1,\ldots,\mathbf{e}_T\}\)，\(\mathbf{e}_i \in \mathbb{R}^k\)。先对嵌入做平均：

\[
\mathbf{h}_0 = \frac{1}{T}\sum_{i=1}^{T} \mathbf{e}_i.
\]

再送入多层感知机：

\[
\mathbf{h}_1 = \phi(\mathbf{W}_1 \mathbf{h}_0 + \mathbf{b}_1), \quad \hat{y} = \sigma(\mathbf{W}_2 \mathbf{h}_1 + \mathbf{b}_2),
\]

其中 \(\phi(\cdot)\) 为 ReLU。损失为交叉熵。超参包括嵌入维度 \(k\)、隐藏层层数与宽度等。

**GloVe + CNN**  
给定嵌入矩阵 \(\mathbf{E} \in \mathbb{R}^{T\times k}\)，施加一维卷积：

\[
\mathbf{c}_j = \operatorname{ReLU}(\mathbf{W}_j * \mathbf{E} + \mathbf{b}_j),
\]

再 max-pooling：

\[
\mathbf{h} = \max_j \mathbf{c}_j.
\]

池化特征经线性层：

\[
\hat{y} = \sigma(\mathbf{w}^\top \mathbf{h} + b).
\]

损失为交叉熵。参数包括卷积核大小、滤波器个数、嵌入维度 \(k\) 等。

**BERT**  
对分词输入 \(x\)，BERT 产生上下文表示：

\[
\mathbf{h} = \mathrm{BERT}(x).
\]

预测为

\[
\hat{y} = \mathrm{softmax}(\mathbf{W}\mathbf{h} + \mathbf{b}).
\]

损失为交叉熵。超参包括学习率 \(\eta\)、批大小、训练轮数等。

**RoBERTa**  
与 BERT 类似，得到上下文表示：

\[
\mathbf{h} = \mathrm{RoBERTa}(x),
\]

并计算：

\[
\hat{y} = \mathrm{softmax}(\mathbf{W}\mathbf{h} + \mathbf{b}).
\]

损失为交叉熵。超参包括学习率 \(\eta\)、批大小、训练轮数等。

### 3.2 分析：理解表示在短文本分类中的角色

为更好理解表示对性能的影响，我们考察以下研究问题：

- 在短文本分类中，**上下文嵌入模型**是否显著优于 TF-IDF 等传统方法？
- **更简单的架构**（如 CNN）是否足以捕捉新闻标题等短文本中的有意义模式？

据此提出假设：

- **H1：** 上下文嵌入模型（如 BERT）因能捕捉更丰富的语义与上下文信息，应优于传统方法。
- **H2：** 对短文本而言，由 CNN 捕捉的**局部 n-gram 特征**可与更复杂的序列模型相竞争。
