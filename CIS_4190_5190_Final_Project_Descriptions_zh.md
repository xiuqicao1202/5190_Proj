## 2. Image2GPS

### 2.1 问题定义与动机

本方向要求训练一个回归模型：输入图像，输出 GPS 坐标。  
你需要先构建图像与 GPS 标签配对的数据集，再训练监督式视觉模型估计拍摄位置。

**任务范围**：目标是预测宾大校园内图像位置。测试区域限定在 33rd & Walnut 到 34th & Spruce 之间的矩形区域（见图 1）。

#### 2.1.1 评估指标

- **平均 Haversine 距离（米）**，越低越好  
- 公式使用地球半径 `R = 6,371,000` 米，按样本平均距离进行报告

### 2.2 代表性数据样例

提供了部分测试数据示例，可用于验证；**不建议加入训练集**。

### 2.3 基线模型训练代码

提供了一个略优于“输出经纬度均值”这种朴素方案的基线 notebook。核心做法：

- 图像缩放到 `224 x 224` 并标准化  
- 对经纬度输出做均值方差归一化（预测后需还原）  
- 主干网络：`ResNet-18`，最后输出层改为 2 维  
- 损失：MSE；优化器：Adam（lr=0.001）；学习率调度器按 epoch 衰减  
- 训练约 10-15 个 epoch 获得基线性能

你们的模型应至少达到该基线水平。  
提醒：Colab 请启用 GPU；免费账户可能遇到算力配额不足。

### 2.4 其他资源

#### 2.4.1 数据采集

你需要采集带 GPS 标签的图像数据。重点是：

- 保证数据质量与采集一致性  
- 注意光照、天气等分布因素  
- 测试集只包含步道上拍摄图像，不必覆盖校园每一处  
- 同一 GPS 点可能有多个视角：建议在同一点位旋转拍摄多张（通常 8 个方向）  
- 拍摄时避免变焦，手机保持竖直

#### 2.4.2 后处理与 GPS 提取

可通过图像 EXIF 信息提取 GPS（经纬度）：

- 需在手机设置中启用相机位置/定位标签  
- iPhone：设置 -> 隐私 -> 定位服务 -> 相机允许定位  
- Android（如 Samsung）：相机设置中开启 Location/Location tags  
- 在大量采集前先确认 EXIF 正常记录

课程提供 notebook，可提取 EXIF 中经纬度并写入 CSV，你可以复制后按需修改，用于生成训练标签。

## 4. 图像风格迁移（Image Style Transfer）

### 4.1 问题定义与动机

你将探索在环境条件变化下的图像风格迁移：  
给定宾大校园场景源图像与目标条件，生成同一场景在目标条件下的新图像，同时保持场景身份、几何结构与语义内容。

关注两类条件维度：

- **时间**：白天 / 日落 / 夜晚  
- **天气**：晴天 / 多云 / 雨天

示例：

- 输入：晴朗白天的 Locust Walk 照片  
- 目标条件：夜晚 + 下雨  
- 输出：同一场景在夜雨条件下的生成图

核心挑战：改变外观而不改变场景本体（建筑、道路、树木等保持一致，只编辑光照、天空与天气效果）。

该项目开放度高，你需要自行决定：

- 数据采集/整理方式  
- 目标条件表示方法  
- 建模方法  
- 评估协议

允许多种方法：VAE、GAN、Diffusion 或其他生成式方法；可尝试成对/非成对训练、检索式思路、条件编码策略、轻量适配等。  
雪、雾、季节变化可作为可选扩展，不是核心要求。  
这是一个更进阶、面向开放问题探索的项目。

#### 4.1.1 性能指标

评估需同时衡量：

1. 生成图是否符合目标条件  
2. 是否保留底层场景内容

课程组将在封闭测试集上使用两个主指标：

- **LPIPS（↓）**：生成图与真实目标图的感知相似性，越低越好  
- **Condition Accuracy（↑）**：固定条件分类器预测生成图条件与目标条件一致的比例，越高越好

二者共同对应项目两大目标：

- 达到目标条件  
- 生成结果在感知上合理且接近真实目标图

补充指标（可选）：

- SSIM（结构相似）  
- PSNR（像素级相似）  
- CLIP Score（图文/图条件对齐）  
- 人工评估（真实性、场景保持、编辑质量）

报告中应说明所用补充指标、合理性及其额外洞见，并鼓励提供定性对比、优缺点与失败案例。  
课程评分主要基于官方两项指标，而非单一主观总分；优秀项目还应体现实验设计严谨性、可复现性与分析深度。

### 4.3 其他资源

#### 4.3.1 数据采集协议

建议尽量标准化采集：

1. 在不同条件下回到相同地点与视角  
2. 避免过度变焦变化  
3. 保持手机/相机朝向一致  
4. 尽量保持同场景构图一致  
5. 避免大量近距离人脸或隐私敏感内容  
6. 记录每张图的大致时间与天气条件

相比大量无关图像，**同一视角在多条件下的一致数据更有价值**。

#### 4.3.2 数据标注与元数据

每张图应附带足够元数据用于训练与评估。至少建议包含：

- 位置/视角标识  
- 时间标签  
- 天气标签

#### 4.3.3 计算资源建议

不要求从零训练大规模生成模型。鼓励基于预训练模型进行高效实验，如：

- adapter  
- 消融实验  
- 轻量微调

#### 4.3.4 可复现性要求

最终提交应包含：

1. 训练/推理代码  
2. 模型权重或 adapter checkpoint（如适用）  
3. 结果复现实验说明  
4. 标注清晰的定性示例  
5. 评估代码与指标结果
